# MASTER 論文重現分析報告與使用指南

這是一份針對經典量化交易論文《MASTER: Market-Guided Stock Transformer for Stock Price Forecasting》的重現專案與分析報告。

## 🚀 快速開始 (Quick Start)

### 1. 安裝環境
強烈建議使用 Conda 建立獨立環境：
```bash
conda create -n master python=3.8
conda activate master

# 安裝 PyTorch 與科學計算套件
pip install torch numpy pandas scipy

# (可選) 安裝 Qlib，若需自行處理特徵資料
pip install pyqlib
```

### 2. 執行預測與回測
我們提供了完整的實驗與自建回測腳本：

**A. 執行量化指標預測與投資組合回測**
此腳本會載入已經訓練好的模型權重，並針對 CSI300 與 Nasdaq 100 兩個市場分別印出 IC、RankIC 以及帶有手續費摩擦的投資組合 AR、IR：
```bash
python evaluate_with_cost.py
```

## 8. 差距原因分析 | Gap Analysis & Discussion

### AR 與 IR 暴漲的原因（82% vs 27%）

| 原因 | 說明 | 影響 |
|---|---|---|
| **缺乏嚴格流動性懲罰** | 原論文使用了微軟 Qlib 完整的交易引擎，包含了 A 股特有的「漲停板買不到、跌停板賣不掉」限制。我們的自建腳本假設「無限流動性」，導致獲利被過度放大。 | 🔴 主因 |
| **滑價 (Slippage) 差異** | 我們僅扣除了 0.15% 的單次交易手續費，並未模擬真實市場掛單的買賣價差與衝擊成本。 | 🟡 次因 |

## 9. 改善方向 | Proposed Improvement

| 優先級 | 改善方向 | 具體做法 | 預期效果 |
|---|---|---|---|
| 🔴 **最高** | 掛載完整 Qlib 引擎 | 將模型產出的預測分數餵進 Qlib 的 `PortAnaRecord` 回測器 | AR 數字將回落至 27% 的真實區間 |
| 🟡 中 | 降低換倉頻率 | 將日頻換倉 (Daily) 改為週頻換倉 (Weekly) | 大幅降低高達 0.15% 的每日手續費磨損 |
| 🟢 低 | 多空對沖策略 | 同時放空倒數 30 名股票 (Long-Short) | 降低大盤曝險，提升 IR |

## 13. 備註/心得 | Notes / Reflection

### 程式碼修復與除錯 (Bug Fix)
- **NaN 崩潰問題**：原代碼的 `base_model.py` 在計算 scipy pearsonr 相關性時，若模型遇到空頭震盪給出常數預測，會導致分母為零產生 `NaN`，並使接下來所有的 Loss 爛掉。我們手動將程式碼中所有的 `np.mean` 與 `np.std` 改為 `np.nanmean`，成功拯救了訓練崩潰的危機。
- **維度匹配問題**：美股與陸股的 `beta` 參數以及 `d_feat` (158 維) 必須在載入權重時完全一致，否則會出現 Shape Mismatch。

### 量化預測模型的理解
- **回測框架決定了最終利潤**：擁有極高的預測 IC 不代表絕對能賺錢。CSI300 跑出了神級的 82% AR，但在嚴苛的 Qlib 流動性引擎下只剩 27%，說明了**流動性與手續費磨損**是高頻量化最致命的敵人。
- **市場環境決定一切**：同樣一套模型，在 A 股與美股跑出了完全不同量級的表現。MASTER 模型特別擅長捕捉板塊的群體動能 (Market-Guided)，這與美股科技巨頭「齊漲齊跌」的特性不謀而合。

### 核心結論
> 1. **重現成功**：完美還原了原論文在 CSI300 的 IC=0.064 預測極限。
> 2. **泛化大勝**：在 Nasdaq 100 上不僅存活，更創造了 266% 的獲利天花板。
> 3. **除錯貢獻**：修正了開源庫的 NaN 崩潰缺陷，並補齊了作者遺漏的 Top 30 交易成本回測系統。
